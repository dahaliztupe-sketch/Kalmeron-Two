# Global Rules — Kalmeron AI

أنت تعمل على منصّة إنتاج عربيّة (Next.js 16 + Firebase + Stripe). كل تغيير يلمس مستخدمين حقيقيّين. اعمل بدقّة الجرّاح، لا بسرعة الـ scaffolder.

## 1. اقرأ قبل ما تعدّل
1. **اقرأ `AGENTS.md` في الجذر** — هو الدستور.
2. **اقرأ `replit.md`** — يحوي تاريخ القرارات + الفخاخ المعروفة.
3. **اقرأ `docs/decisions/`** قبل تغيير أيّ قرار معماري.
4. **اقرأ `docs/THREAT_MODEL.md`** قبل لمس أيّ شيء أمني.
5. **اقرأ توثيق Next.js المحلّي**: `node_modules/next/dist/docs/` — هو المرجع الرسمي.

## 2. لغة العمل
- المستخدم الأساسي يتحدّث **العربيّة المصريّة**. ردّ عليه بنفس اللغة بأسلوب مهني هادئ.
- النصوص الموجَّهة للمستخدم النهائي في الـ UI: **عربيّة فصحى مبسّطة + RTL**.
- الـ commits + PR descriptions + كود التعليقات: عربي أو إنجليزي حسب الموجود في الملف.

## 3. أسلوب التغييرات
- **لا تُعِد كتابة من الصفر** ما لم يكن الملف معطوباً تماماً. أصلح، لا تستبدل.
- **التغييرات الذرّيّة:** PR صغير يحلّ مشكلة واحدة أحسن من PR ضخم.
- **حافظ على الهيكل القائم.** لو الملف بـ `kebab-case`، استمرّ. لو الـ exports بـ `named`، لا تُحوّل لـ `default`.
- **اختبر كل تغيير محلّيّاً** قبل التسليم: `npm run typecheck && npm run lint && npm run test`.

## 4. ممنوعات صارمة
- ❌ لا `as any` جديدة في `src/`, `app/`, `lib/`, `components/`.
- ❌ لا queries على Firestore بدون `.limit()`.
- ❌ لا قواعد `if true` في `firestore.rules`.
- ❌ لا secrets في الكود؛ استخدم `process.env.X` (server) أو `NEXT_PUBLIC_X` (client) فقط.
- ❌ لا `console.log` تركها بعد debug؛ استخدم `pino` (`src/lib/logger.ts`).
- ❌ لا `Math.random()` في JSX render — استخدم `useMemo` أو حسبها مرّة.
- ❌ لا تُحوّل API route من Node إلى Edge بدون التحقّق من `firebase-admin`/`pdf-parse`.
- ❌ لا تُنشئ `README.md` ملخّصة للمشروع تلقائيّاً.

## 5. أوامر التحقّق (إلزاميّة قبل التسليم)
```bash
npm run typecheck   # 0 errors
npm run lint        # 0 errors
npm run test        # كل الاختبارات تمرّ
```

## 6. حوار مع المستخدم
- إذا كان الطلب غامضاً، اسأل **سؤالاً واحداً** قبل التنفيذ.
- إذا كنت ستحذف/تغيّر شيئاً مهماً، **اشرح الأثر بجملة واحدة** قبل التنفيذ.
- لا تستخدم emojis في الردّ على المستخدم ما لم يستخدمها هو أوّلاً.

## 7. تتبّع الذاكرة
- بعد كل جلسة عمل ذات أثر، **حدّث `replit.md`** بقسم Recent Updates يصف: ماذا، لماذا، الملفّات، وأمر التحقّق.
- لا تحذف أو تختصر أقسام `replit.md` التاريخيّة بدون إذن.
