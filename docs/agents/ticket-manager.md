# Agent System Card — Ticket Manager (مدير تذاكر الدعم)

**Version:** 1.0 · **Last reviewed:** 2026-05-07 · **Owner:** Support Pod

## 1. Purpose
يُساعد فرق دعم العملاء في الشركات الناشئة المصرية على إدارة التذاكر بكفاءة: تصنيف التذاكر بالنوع والأولوية، كتابة ردود دعم احترافية بالعربية، والتحليل الدوري لأنماط التذاكر لتحسين المنتج والعملية.

## 2. Capabilities
- تصنيف التذاكر (Bug, Feature Request, How-to, Billing, Complaint, Outage) وتحديد الأولوية (P0-P3).
- كتابة ردود دعم احترافية بالعربية المصرية وفق هيكل الرد المثالي.
- قوالب ردود جاهزة لكل نوع تذكرة.
- توثيق الحلول وتحويلها لمقالات في قاعدة المعرفة.
- تحليل أنماط التذاكر المتكررة وتحديد أولويات المنتج.
- قياس SLA Compliance وتقديم توصيات تحسين.

## 3. Out-of-scope
- لا يصل مباشرة لنظام Ticketing (Freshdesk, Zendesk) للمستخدم.
- لا يُغلق تذاكر المستخدمين الفعليين تلقائياً.
- لا يُعالج مشاكل تقنية معقدة بدل الفريق الهندسي.

## 4. Risk class — **Limited**
أداة دعم واتصال استرشادية.

## 5. Data touched
| Data | Source | Stored? | Retention |
|---|---|---|---|
| نماذج تذاكر (لا بيانات شخصية للمستخدمين) | إدخال | session فقط | تُحذف نهاية الجلسة |

## 6. Tools available
`rag.search` (intent: `support_operations`).

## 7. Known failure modes
- ردود دعم رسمية جداً تفتقر للدفء في ثقافة الخدمة المصرية.
- تصنيف خاطئ لتذاكر متعددة الأبعاد.
- SLA توصيات لا تراعي حجم فريق الدعم في الشركات الصغيرة.

## 8. Evaluation
- Golden set: 30 نموذج تذكرة مع تقييم جودة الرد.
- عتبة قبول: 0.83.

## 9. Human-in-the-loop
تذاكر P0 (انقطاع الخدمة) تُصعَّد لفريق بشري فوراً.

## 10. Disclosures
- "الردود المُولَّدة قوالب. كيّفها لتعكس الحالة الفعلية للعميل قبل الإرسال."
