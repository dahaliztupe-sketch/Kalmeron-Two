# Agent System Card — Data Privacy Agent (وكيل خصوصية البيانات)

**Version:** 1.0 · **Last reviewed:** 2026-05-07 · **Owner:** Legal Eng + Counsel of record

## 1. Purpose
يُرشد الشركات الناشئة المصرية على الامتثال لقانون حماية البيانات الشخصية 151/2020 ولوائحه التنفيذية، بما يشمل: تصنيف البيانات، بناء سياسات الخصوصية، إدارة موافقات المستخدمين، وتقييم مخاطر البيانات.

## 2. Capabilities
- شرح اشتراطات قانون 151/2020 ولوائحه.
- توليد مسودات سياسة الخصوصية وإشعارات الموافقة.
- بناء Data Inventory وتصنيف البيانات الحساسة.
- توجيه إجراءات الاستجابة لاختراقات البيانات (Breach Response).
- مراجعة آليات الموافقة (Consent Mechanisms).
- إعداد Data Processing Agreements (DPA) مع الأطراف الثالثة.
- توجيه متطلبات نقل البيانات خارج مصر.

## 3. Out-of-scope
- **لا يُقدّم استشارة قانونية رسمية** ولا يحلّ محلّ مستشار حماية بيانات مرخّص (DPO).
- لا يُنفّذ تدابير أمنية تقنية مباشرة.
- لا يمثل الشركة أمام المجلس الأعلى لتنظيم الاتصالات.

## 4. Risk class — **High**
تفسير قانوني لتشريعات حماية البيانات مع تبعات قانونية لعدم الامتثال. يلزم:
- إفصاح صريح قبل كل جلسة.
- audit log لكل توصية.
- DPIA منشور في `docs/dpia/data-privacy-dpia.md`.

## 5. Data touched
| Data | Source | Stored? | Retention |
|---|---|---|---|
| وصف أنظمة بيانات الشركة | إدخال | في `legal_drafts/{id}` | حتى الحذف |
| مسودات سياسات الخصوصية | مولَّد | في `legal_drafts/{id}` | حتى الحذف |

## 6. Tools available
`rag.search`, `legal.template` (intent: `privacy_compliance`).

## 7. Known failure modes
- الخلط بين اشتراطات GDPR الأوروبي والقانون المصري 151/2020 (متشابهان لكن ليسا متطابقَين).
- إغفال اللوائح التنفيذية الصادرة بعد القانون الأساسي.
- توصيات Data Residency لا تراعي البنية التحتية السحابية المستخدمة.

## 8. Evaluation
- Golden set: 25 سيناريو خصوصية بيانات.
- عتبة قبول: 0.88.

## 9. Human-in-the-loop
كل سياسة خصوصية مُولَّدة تُعلَّم "DRAFT" وتنتظر مراجعة مستشار قانوني.
