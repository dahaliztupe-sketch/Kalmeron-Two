[CONTEXT]
نحن نعمل على منصة "كلميرون تو" (Kalmeron Two)، وهي نظام تشغيل متكامل لرواد الأعمال مبني على Next.js 16، TypeScript، وFirebase. تضم المنصة 16 وكيلاً متخصصاً في 8 أقسام، مع بنية تحتية متقدمة تشمل Neo4j، RAG ذاتي التصحيح، نظام دفاع متعدد الطبقات، نظام فوترة، بيئة تنفيذ أكواد، توأم رقمي، وذكاء طرفي.

لقد وصلنا إلى مرحلة حرجة تتطلب مراجعة شاملة وعميقة للمشروع بأكمله، وتقييم جاهزيته للإنتاج، ووضع خطة تطوير استراتيجية للمرحلة القادمة. نريد منك أن تعمل **كفريق من الخبراء المتخصصين** (Panel of Experts) لتقديم تحليل شامل ومتعدد الأبعاد.

[YOUR MISSION]
أنت فريق من الخبراء الافتراضيين. قم بإجراء مراجعة شاملة للمشروع، ثم قدم خطة تطوير استراتيجية. اعمل بشكل منهجي ومنظم. استخدم أحدث المعايير وأفضل الممارسات لعام 2026.

## 📋 المرحلة 1: التدقيق الشامل (Comprehensive Audit)

قم بفحص المشروع بدقة من خلال الزوايا التالية. **لا تقم بتعديل أي كود. فقط حلل وقيم.**

### 1.1 التقييم الاستراتيجي (الخبراء: استراتيجي النمو، محلل السوق، المخطط المالي، خبير الاستدامة)
- هل تحل المنصة مشكلة حقيقية لرائد الأعمال المصري؟
- كيف نقارن بالمنافسين العالميين (Denovo، Spiritt.io، Jasper AI، Launchium)؟
- ما هي الميزة التنافسية الأقوى لدينا؟ وما هي أضعف نقاطنا؟
- هل نموذج العمل (Freemium + اشتراكات) قابل للاستمرار؟
- ما هي نقطة التعادل المتوقعة؟ وما هو معدل الحرق الشهري؟

### 1.2 التدقيق التقني (الخبراء: مهندس البرمجيات الرئيسي، مهندس DevOps، مهندس قاعدة البيانات، مهندس الذكاء الاصطناعي، مهندس الواجهة الأمامية، مهندس تطبيقات الهاتف المحمول، مهندس الحوسبة الطرفية)
- جودة الكود: هل هناك ديون تقنية؟ هل TypeScript صارم؟ هل هناك استخدام لـ `any`؟
- هيكل المشروع: هل يتبع أفضل ممارسات Next.js 16 (Server Components، Caching، Turbopack)؟
- أداء قاعدة البيانات: هل Firestore مهيأ بشكل صحيح؟ هل Neo4j يعمل؟ هل هناك استعلامات غير فعالة؟
- أداء الوكلاء: هل جميع الوكلاء يستجيبون بشكل صحيح؟ هل هناك أي وكيل معطل أو بطيء؟
- RAG: هل نظام RAG ذاتي التصحيح يعمل بشكل صحيح؟ هل دقة الاسترجاع عالية؟
- واجهة المستخدم: هل RTL يعمل بشكل صحيح في جميع الصفحات؟ هل التصميم موحد؟
- تطبيق الهاتف المحمول: هل Expo يعمل بشكل صحيح؟ هل وضع عدم الاتصال فعال؟
- الحوسبة الطرفية: هل WebGPU يعمل؟ هل النماذج المحلية تحمل بشكل صحيح؟

### 1.3 التدقيق الأمني (الخبراء: مهندس الأمان، مختبر الاختراق، مدقق الامتثال للذكاء الاصطناعي، مدقق الامتثال للخصوصية، مدقق الحوكمة، مدقق أمان الهاتف المحمول، مدقق الامتثال المالي)
- هل قواعد Firestore آمنة (لا يوجد `allow read, write: if true;`)؟
- هل جميع مسارات API محمية بشكل صحيح؟
- هل هناك حماية ضد Prompt Injection؟ هل ClawGuard/PlanGuard مفعلان؟
- هل مفاتيح API مخزنة بشكل آمن (متغيرات بيئة، وليس في الكود)؟
- هل هناك ثغرات CSRF، XSS، أو تسرب بيانات؟
- هل المنصة متوافقة مع EU AI Act؟ مع GDPR؟ مع قانون حماية البيانات المصري؟
- هل مسارات التدقيق (Audit Trails) تعمل بشكل صحيح؟
- هل تطبيق الهاتف المحمول آمن؟ هل المصادقة البيومترية مطبقة بشكل صحيح؟

### 1.4 تحسين التكلفة والأداء (الخبراء: خبير تحسين التكلفة، خبير قابلية التوسع، محلل الأداء)
- هل استراتيجية توجيه النماذج (Model Routing) فعالة؟ كم نسبة التوفير المتوقعة؟
- هل هناك استخدام للتخزين المؤقت (Caching)؟ هل يمكن تحسينه؟
- هل Firestore مهيأ بشكل صحيح لتقليل التكاليف؟
- هل هناك استعلامات N+1 أو استهلاك مفرط للموارد؟
- ما هي Core Web Vitals؟ هل هناك مجال للتحسين؟

### 1.5 تجربة المستخدم وإمكانية الوصول (الخبراء: مصمم تجربة المستخدم، خبير إمكانية الوصول، خبير التعريب والتوطين، خبير تحسين محركات البحث)
- هل تدفق المستخدم (من التسجيل إلى أول قيمة) سلس؟
- هل المنصة متوافقة مع WCAG 2.2؟
- هل RTL يعمل بشكل صحيح في جميع الصفحات؟
- هل المحتوى المختلط (عربي/إنجليزي) يعرض بشكل صحيح؟
- هل Metadata و Sitemap و Robots.txt مهيأة بشكل صحيح؟

### 1.6 العمليات والتطوير (الخبراء: مدقق عمليات التطوير، خبير الاختبار، خبير التوثيق، خبير الأتمتة)
- هل عملية Git منظمة؟ هل هناك استراتيجية فروع واضحة؟
- هل هناك اختبارات وحدة و E2E كافية؟
- هل التوثيق كامل ومحدث؟
- هل هناك أتمتة كافية للاختبار والنشر؟
