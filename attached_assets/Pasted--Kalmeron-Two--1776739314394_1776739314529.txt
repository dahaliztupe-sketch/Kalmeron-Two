## 🛡️ المرحلة P0 (حرجة - فورية): تدقيق قواعد أمان Firestore

### المهمة 2.1: كتابة قواعد أمان صارمة
- راجع ملف `firestore.rules`. استبدل أي قواعد فضفاضة (مثل `allow read, write: if true;`) بالقواعد الآمنة التالية:
- **مجموعة `users`**:
```

match /users/{userId} {
allow read: if request.auth != null && request.auth.uid == userId;
allow create: if request.auth != null && request.auth.uid == userId;
allow update: if request.auth != null && request.auth.uid == userId;
}

```
- **باقي المجموعات (`ideas`, `business_plans`, `chat_history`, `user_memory`, إلخ)**:
```

match /{collection}/{document} {
allow read, write: if collection in ['ideas', 'business_plans', 'chat_history', 'user_memory', 'saved_companies', 'mistakes_viewed', 'personas', 'market_experiments']
&& request.auth != null
&& request.auth.uid == resource.data.userId;
}

```
- **مجموعات عامة (مثل `opportunities`)**:
```

match /opportunities/{doc} {
allow read: if true;
allow write: if false;
}

```
- **اختبار القواعد**: استخدم Firebase Emulator Suite أو Rules Playground للتحقق من فعاليتها.

- في `src/app/api/chat/route.ts` و `src/app/api/orchestrator/route.ts`، قم بتغليف المعالج بـ `withRateLimit`:
