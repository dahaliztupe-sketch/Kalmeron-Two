## المهمة 1: إصلاح زر "التسجيل باستخدام Google" في `/auth/signup`

### 1.1 تشخيص المشكلة
- الزر لا يستدعي وظيفة المصادقة. ربما لا يوجد `onClick` handler، أو أن المنطق غير مكتمل.

### 1.2 الإصلاح المطلوب
- افتح `src/app/auth/signup/page.tsx`.
- تأكد من وجود هذه الاستيرادات:
  ```typescript
  import { getAuth, signInWithPopup, GoogleAuthProvider } from 'firebase/auth';
  import { useRouter } from 'next/navigation';
```

· أضف دالة handleGoogleSignUp:
  ```typescript
  const handleGoogleSignUp = async () => {
    const auth = getAuth();
    const provider = new GoogleAuthProvider();
    try {
      const result = await signInWithPopup(auth, provider);
      const isNewUser = result.user.metadata.creationTime === result.user.metadata.lastSignInTime;
      if (isNewUser) {
        router.push('/onboarding');
      } else {
        router.push('/dashboard');
      }
    } catch (error) {
      console.error('Google sign-up error:', error);
      // يمكنك إضافة Toast للإبلاغ عن الخطأ
    }
  };
  ```
· اربط هذه الدالة بزر "التسجيل باستخدام Google" عبر onClick={handleGoogleSignUp}.

المهمة 2: إنشاء صفحة تسجيل الدخول (/auth/login)

2.1 المشكلة

· رابط "تسجيل الدخول" في صفحة التسجيل يؤدي إلى 404.

2.2 الإصلاح المطلوب

· أنشئ الملف src/app/auth/login/page.tsx.
· استخدم نفس تصميم صفحة التسجيل تقريبًا (يمكنك نسخ الكود وتعديله).
· أضف زر "تسجيل الدخول باستخدام Google" مع دالة handleGoogleLogin:
  ```typescript
  const handleGoogleLogin = async () => {
    const auth = getAuth();
    const provider = new GoogleAuthProvider();
    try {
      const result = await signInWithPopup(auth, provider);
      // تحقق مما إذا كان الملف الشخصي مكتملاً
      const userDoc = await getDoc(doc(db, 'users', result.user.uid));
      if (userDoc.exists() && userDoc.data().profileCompleted) {
        router.push('/dashboard');
      } else {
        router.push('/onboarding');
      }
    } catch (error) {
      console.error('Google login error:', error);
    }
  };
  ```
· أضف رابط "ليس لديك حساب؟ سجل الآن" يوجه إلى /auth/signup.

المهمة 3: إنشاء صفحة إكمال الملف الشخصي (/onboarding)

3.1 المشكلة

· بعد التسجيل، لا يوجد توجيه إلى صفحة إكمال الملف الشخصي.

3.2 الإصلاح المطلوب

· تأكد من وجود src/app/onboarding/page.tsx. إذا لم يكن موجودًا، أنشئه.
· اجعل الصفحة محمية (لا يمكن الوصول إليها إلا بعد تسجيل الدخول):
  ```typescript
  useEffect(() => {
    const unsubscribe = onAuthStateChanged(auth, (user) => {
      if (!user) router.push('/auth/login');
    });
    return () => unsubscribe();
  }, []);
  ```
· أنشئ نموذجًا لجمع: startupStage (قائمة منسدلة)، industry (قائمة منسدلة)، governorate (قائمة منسدلة)، و name (اختياري).
· عند الإرسال، احفظ البيانات في Firestore:
  ```typescript
  await setDoc(doc(db, 'users', user.uid), {
    name: formData.name,
    startupStage: formData.startupStage,
    industry: formData.industry,
    governorate: formData.governorate,
    profileCompleted: true,
    createdAt: new Date(),
  }, { merge: true });
  router.push('/dashboard');
  ```

المهمة 6: اختبار التدفق الكامل

6.1 سيناريو الاختبار

1. امسح بيانات المتصفح (أو استخدم وضع التصفح المتخفي).
2. اذهب إلى /.
3. اضغط على "ابدأ الآن" أو "تسجيل الدخول" (يجب أن يوجه إلى /auth/login).
4. اضغط على "ليس لديك حساب؟ سجل الآن" (يجب أن يوجه إلى /auth/signup).
5. اضغط على "التسجيل باستخدام Google". اختر حساب Google.
6. يجب أن تنتقل إلى /onboarding.
7. املأ البيانات واحفظ. يجب أن تنتقل إلى /dashboard.
8. اضغط على زر تسجيل الخروج. يجب أن تنتقل إلى /.
9. اذهب إلى /auth/login. اضغط على "تسجيل الدخول باستخدام Google". يجب أن تنتقل مباشرة إلى /dashboard.
10. اكتب مسارًا غير موجود (مثل /xyz). يجب أن تظهر صفحة 404، وعند الضغط على "العودة للوحة التحكم" يجب أن تنتقل إلى /dashboard.

6.2 التحقق

· تأكد من عدم وجود أخطاء في وحدة تحكم المتصفح.
· تأكد من حفظ بيانات المستخدم بشكل صحيح في Firestore.
