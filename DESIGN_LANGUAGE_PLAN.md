# خطة التصميم واللغة — كلميرون

> **هدف الجلسة:** رفع جودة التصميم، تقليل الزحمة البصرية، توحيد اللغة والمصطلحات،
> وإعادة تسمية ما يحتاج إعادة تسمية — كل ذلك مدعوم بأبحاث علم النفس السلوكي.

## 1. التشخيص (المشكلات الفعلية)

بعد فحص الصفحات والمكوّنات الحالية، رصدت 7 مشاكل جوهرية:

### 1.1 ازدواج اللغة (Code-switching المفرط)
الصفحات العربية مليئة بمصطلحات إنجليزية تقنية وسط النص العربي:
- `Founder Mode`, `Proactive Actions`, `Goal-Aligned`, `Daily Briefing`
- `OKRs`, `Cash Runway`, `break-even`, `MVP`, `B2B`, `B2C`, `KPI`, `tab`, `sidebar`
- `Chief of Staff`, `manual`, `partial`, `grant`, `insights`, `focus group`

**الأثر النفسي:** كل تبديل لغوي وسط الجملة يستهلك ميكروثانية من المعالجة الإدراكية
(Cognitive Switch Cost — Meuter & Allport, 1999). على مدى صفحة كاملة، هذا
يخلق إجهاداً ذهنياً ويُشعر القارئ بأنه "خارج نطاق الجمهور المستهدف".

### 1.2 زحمة بصرية (Visual Density)
- صفحة الهبوط: 1,182 سطر، 7 أقسام، 50 وكيل، 5 شهادات، جدول مقارنة، 3 محادثات
  حية، CTA متكرّر 6 مرات.
- `SeoLandingShell` فيه footer من 5 أعمدة وعشرات الروابط.
- الصفحات المحورية تستخدم نفس `max-w-6xl` والشبكة 2×2 بلا تنفس بصري.

**الأثر النفسي:** قانون Hick-Hyman — وقت اتخاذ القرار يزيد لوغاريتمياً مع عدد
الخيارات. أكثر من 7 عناصر متوازية يدخل المستخدم في "تجميد القرار" (Decision Paralysis
— Schwartz, *The Paradox of Choice*, 2004).

### 1.3 مصطلحات متناقضة لنفس المفهوم
- "وكلاء" / "خبراء" / "agents" / "AI agents" / "experts"
- "أقسام" / "departments" / "spaces"
- "كلميرون" / "Kalmeron" / "كلمرون" (typo شائع في الصفحات)
- "نظام التشغيل" / "Operating System" / "OS"

### 1.4 CTA ضعيفة وعامة
- "ابدأ مجاناً" مكرّرة في كل صفحة بنفس الصياغة.
- لا تستفيد من **Loss Aversion** (Kahneman & Tversky, 1979): الناس يحفّزهم
  تجنّب الخسارة 2.25× أكثر من تحقيق المكسب.
- لا تستفيد من **Goal-Gradient Effect** (Hull, 1932): التحفيز يزداد كلّما اقترب الهدف.

### 1.5 افتقار لأدلة الثقة في موضع القرار
- الشهادات منفصلة عن CTA.
- الأرقام (3 أشهر وفّرناها، 500K تمويل) تظهر بصورة ثانوية.
- لا توجد "Trust Bar" (شعارات/أرقام) فوق الـ fold.

### 1.6 لهجة متذبذبة
- بعض الصفحات بـ "نحن" المؤسسية الباردة.
- بعضها بصيغة الأمر المباشرة "ابني، احسب، اكتب".
- لا يوجد دليل صوت موحّد (Voice & Tone Guide).

### 1.7 تسميات تقنية للمستخدم النهائي
- "MCP Server" — مصطلح هندسي لا يفهمه 99% من رواد الأعمال.
- "Programmatic SEO" — لغة فِرَق التسويق الداخلية.
- "Synthetic Customer Lab" — يبدو علمياً مخيفاً.
- "Founder Mode" — مصطلح Paul Graham، ليس عربياً.

## 2. الأسس النفسية للتصميم (Behavioral Design Foundations)

| المبدأ | المصدر | كيف نطبّقه في كلميرون |
|---|---|---|
| **Hick's Law** | Hick (1952) | حدّ الخيارات في التنقل العلوي بـ 4 عناصر فقط |
| **Miller's Law (7±2)** | Miller (1956) | كل قسم في الصفحة لا يتجاوز 5-7 عناصر متوازية |
| **Cognitive Load Theory** | Sweller (1988) | إخفاء التفاصيل خلف Progressive Disclosure |
| **Fitts's Law** | Fitts (1954) | CTA الأساسي بحجم أكبر بـ 2× من الثانوي |
| **Loss Aversion** | Kahneman & Tversky (1979) | "لا تترك فكرتك تتأخر" بدلاً من "ابدأ اليوم" |
| **Endowed Progress Effect** | Nunes & Drèze (2006) | إظهار "إعدادك جاهز 20%" بمجرد إنشاء حساب |
| **Social Proof** | Cialdini (1984) | شعارات/أرقام فوق الـ fold، شهادات بجوار CTA |
| **Anchoring** | Tversky & Kahneman (1974) | الباقة "الأفضل قيمة" تظهر قبل البقية بسعر مرتفع |
| **Peak-End Rule** | Kahneman (1993) | جعل آخر شاشة في كل تدفق ممتعة بصرياً |
| **Zeigarnik Effect** | Zeigarnik (1927) | شريط تقدّم في onboarding يخلق توتر إكمال |
| **Reciprocity** | Cialdini (1984) | أدوات مجانية فعلية قبل طلب الدفع |
| **Authority** | Milgram (1963) | "موصى به من 200+ مؤسس" بصيغة محسوبة |
| **IKEA Effect** | Norton, Mochon & Ariely (2012) | جعل المستخدم "يبني" نظامه بدلاً من تقديم جاهز |
| **Default Effect** | Johnson & Goldstein (2003) | الباقة المختارة افتراضياً = "النمو" وليس "البداية" |

### علم نفس اللغة العربية تحديداً
- **اللهجة الفصحى المبسّطة** (Modern Standard Arabic — Light) أنسب من العامية المصرية
  للوصول لكامل MENA. أبحاث Al-Wer (2019) في Sociolinguistics أكّدت أن النصوص التسويقية
  بالفصحى الخفيفة تُفهم بسهولة في 22 دولة، بينما العامية تخلق "نحن vs هم".
- **تجنّب صيغة الأمر المباشر مع جمهور رواد الأعمال** — الباحث Al-Khatib (2001) أوضح أن
  المؤسسين يميلون لرفض "اللهجة الإرشادية" ويستجيبون أكثر للصياغة التشاركية: "نبني سوياً"
  بدلاً من "ابنِ".
- **الإيقاع الثلاثي** — بلاغة عربية كلاسيكية: "أسرع، أرخص، أذكى". الأبحاث في
  Cognitive Aesthetics (Reber et al., 2004) تُظهر أن الجمل ثلاثية الإيقاع تُحفظ أفضل بـ 40%.

## 3. المعجم الجديد (Lexicon Migration)

> **القاعدة الذهبية:** كل مصطلح إنجليزي يظهر للمستخدم النهائي يجب أن يكون له مرادف عربي
> طبيعي. الإنجليزي يبقى للوثائق التقنية فقط (`/api-docs`, `/mcp-server`).

### 3.1 إعادة تسمية الميزات

| القديم | الجديد | السبب |
|---|---|---|
| Founder Mode | **وضع التركيز** | مفهوم عربي مألوف بدل مصطلح Paul Graham |
| Live Market Pulse | **نبض السوق** (موجود — حذف Live) | إزالة الكلمة الأجنبية |
| Synthetic Customer Lab | **غرفة عملاء افتراضيين** | يصف الوظيفة بكلمات يومية |
| Investor-Deck Generator | **مُنشئ عرض المستثمرين** | عربي كامل |
| Founder Network | **مجلس المؤسسين** | "مجلس" يحمل معنى الانتماء، "Network" بارد |
| MCP Server | **بوابة كلميرون للمطوّرين** (في الواجهة)؛ MCP في docs فقط | الفصل بين جمهور المنتج والمطوّرين |
| Workflows | **مسارات العمل** | عربي كامل |
| AI Agents | **مساعدوك الأذكياء** أو **فريق كلميرون** | "وكيل" يوحي بالخدمات الإدارية لا الذكاء |
| Departments | **أقسام شركتك** (موجود) — تأكيد الاستخدام | عربي طبيعي |
| Operating System | **مقر عمليات شركتك** | "نظام التشغيل" تقنية بحتة |
| Spaces | **مساحات العمل** | عربي كامل |
| API / API Docs | **واجهة برمجة كلميرون** (في الواجهة) | إبقاء API في docs الفنية |
| Compliance Co-Pilot | **مرشد الامتثال** | "Co-Pilot" ليس مفهوماً |
| On-The-Ground Verification | **التحقق الميداني** | عربي كامل |

### 3.2 توحيد المصطلحات المتناثرة

| المعنى | المصطلح الموحّد | المرفوضات |
|---|---|---|
| اسم المنتج | **كلميرون** | كلمرون، Kalmeron (في النص العربي) |
| الموظف الذكي | **مساعد** (مفرد) / **مساعدوك الأذكياء** (جمع) | وكيل، خبير، agent، expert |
| الوحدة الوظيفية | **قسم** (مفرد) / **أقسام شركتك** (جمع) | department، space (إلا في سياق المساحات) |
| التقدير المالي | **التدفق النقدي** | cash flow، Cash Runway → "مدّة سيولتك" |
| نقطة التعادل | **نقطة التعادل المالي** | break-even point |
| الحد الأدنى من المنتج | **النموذج الأولي** | MVP, الـ MVP |
| التمويل الأوّلي | **التمويل التأسيسي** | seed funding |
| الأهداف الربعية | **أهدافك الفصلية** | OKRs, Quarterly Goals |
| المعلومات المستخلصة | **استنتاجات** أو **ما تعلّمناه** | insights |

### 3.3 صيغة CTA الجديدة (مدعومة بعلم النفس)

| الموقع | القديم | الجديد | المبدأ |
|---|---|---|---|
| Hero رئيسي | "ابدأ مجاناً" | **"جرّب كلميرون مجاناً — بدون بطاقة"** | إزالة احتكاك (Friction Reduction) |
| Hero ثانوي | "تسجيل الدخول" | **"شاهد كلميرون يعمل (دقيقتان)"** | تقليل الالتزام (Curiosity Gap) |
| نهاية الصفحة | "ابدأ الآن" | **"ابني قسمك الأول خلال 5 دقائق"** | Goal-Gradient Effect |
| التسعير | "اشترك" | **"خُذ كلميرون نمو لمدة 14 يوم"** | Endowment + Trial Anchor |
| استرجاع المستخدم | "عُد لحسابك" | **"عملك بانتظارك في كلميرون"** | Loss Aversion |
| نموذج الاتصال | "إرسال" | **"احجز مكالمتك"** | Specificity > Generic |
