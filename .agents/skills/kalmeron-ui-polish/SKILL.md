# Kalmeron UI Polish

## When to Use
- When a Kalmeron page looks generic, flat, crowded, or visually inconsistent.
- When improving RTL Arabic-first layout, spacing, typography, motion, or component balance.
- When replacing placeholder/demo-looking UI with production-ready polish.

## Instructions
- Preserve the existing product identity: dark premium surfaces, strong contrast, clear hierarchy, RTL correctness.
- Fix one screen at a time. First clarify hierarchy, then spacing, then copy, then motion.
- Avoid generic AI patterns: identical cards, overused gradients, random glow, and repetitive icon blocks.
- Use motion only where it improves comprehension.
- Prefer concrete, readable Arabic labels and concise helper text.
- Keep forms obvious, safe, and easy to scan.
- If a screen feels unfinished, remove decorative noise before adding more effects.

## Output Checklist
- Clear primary action
- Strong title and supporting copy
- Consistent spacing and alignment
- RTL-safe structure
- Minimal but purposeful motion
- No demo wording in user-facing production paths
