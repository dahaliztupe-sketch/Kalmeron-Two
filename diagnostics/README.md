# Diagnostics

Internal automated reports for this project. Generated by `npm run diag`.

## Layout
- `errors/latest.md` + `latest.json` — all detected errors (TypeScript, ESLint, npm audit, broken imports, Python syntax, recent log errors)
- `gaps/latest.md` + `latest.json` — what's missing or needs to be developed (TODOs, stubs, missing tests, missing translations, missing env vars, missing route fallbacks, undocumented folders, skeleton pages)
- `reports/dashboard.md` + `dashboard.json` — unified health dashboard with score and top priorities
- `history/` — dated snapshots of every run for trend analysis

## Commands
```bash
npm run diag          # full scan: errors + gaps + dashboard
npm run diag:errors   # errors only
npm run diag:gaps     # gaps only
```

## Health score
Computed as `100 - (critical*10 + high*4 + medium*1.5 + low*0.3)` with gaps
weighted at 50%. A score >= 80 is healthy.

## Severity levels
- **critical** — production-breaking, must fix immediately
- **high** — likely bug, security risk, or unfinished feature
- **medium** — code quality, missing tests, missing translations
- **low** — style, missing docs, low priority TODOs

Reports overwrite `latest.*` on each run; previous runs are preserved in
`history/` so you can diff progress over time.
